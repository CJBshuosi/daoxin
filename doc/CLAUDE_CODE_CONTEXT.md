## 一、项目是什么

**道心文案** — 一个面向短视频创作者的 AI 文案生成工具。

**当前形态**：本地单页应用，无需服务器，浏览器直接打开使用。
**目标用户**：运营多个短视频垂类账号的创作者。
**核心价值**：不是"更快生成文案"，而是"生成更好的文案"。

## 三、当前已实现的功能

### Track（赛道）系统
- 用户可自定义新增 / 删除赛道
- 每个 Track 独立存储：名称、颜色、描述、禁忌词、记忆、对标账号
- 所有数据通过 `localStorage` 持久化（key: `daoxin_v1`）

### AI 文案生成
- 浏览器直连 Anthropic API（`claude-sonnet-4-20250514`）
- 输入主题 → 输出：正文文案 + 3条爆款标题 + BGM推荐
- API 返回 JSON 格式：`{copytext, titles, music, memory_update}`

### 记忆系统
- 每次生成后，AI 返回 `memory_update` 字段
- 自动追加到当前 Track 的 `memory` 字段
- 下次生成时注入进 System Prompt
- 用户可手动编辑记忆

### Few-shot 支持
- 每个 Track 可配置爆款参考文案（`fewShot` 字段）
- 注入进 System Prompt，AI 模仿风格生成

### Prompt 组装（`buildSystemPrompt` 函数）
```
角色定义
+ 对标账号
+ 用户偏好记忆（动态注入）
+ Few-shot爆款参考（动态注入）
+ 禁忌词（动态注入）
+ 文案结构规范（钩子→内容→互动引导）
+ JSON输出格式
```

## 四、当前最大的问题（下一步要解决的）

**现在的 Agent 只是单次对话**：输入主题 → 直接输出文案。

这跟直接问 Claude 没有本质区别。

**真正好的文案需要多步骤流程**（来自用户提供的爆款文案指南）：
1. 选题确认 — 这个选题值得做吗？对应什么用户需求？
2. 框架梳理 — 用故事讲述型还是主张论证型？
3. 第一句话 — 用哪种钩子（反常识/漫热点/情感共鸣/痛点）？
4. 底层优化 — 有没有交流感？情绪回路对不对？信息密度够吗？
5. 输出文案

**目标**：把 Agent 改成多步引导流程，而不是一步生成。

## 五、有参考价值的爆款文案知识（用户提供的指南摘要）

### 平台机制（影响生成策略）
- 评分七大指标：完播率、点赞、评论、分享、转粉、主页停留、转化率
- 完播率最重要
- 标签系统：文字、画面、音乐、评论热词都影响标签

### 第一句话（钩子）的6种方法
1. 反常识 — 打破大众已有认知
2. 漫热点 — 蹭热点但要有主题期待
3. 情感共鸣 — 说出观众心里的话
4. 故事之传奇色彩 — 他这样做，我为什么不行
5. 权威性、数字、证据、名人
6. 植入欲望点 — 与我相关

### 让观众点赞的6个机会
1. 过人之处，令人钦佩
2. 真心对你好
3. 情怀传播、正能量
4. 高见！卧槽感
5. 突如其来的刺激、高潮、反转
6. 伴随着笑容笑声点赞

### 底层内核优化
- **交流感**：抖音比其他媒体更加私人的交流模式，用"我、你、我们"
- **信息密度**：建立3秒惯调，段落化文字，有效信息
- **情绪回路**：短句排比增强感染力，句短句长营造节奏感

### 选题确认标准
- 四个诉求：有趣、有共鸣、有价值、有美感
- 人类底层欲望：生存欲、享受欲、收藏欲、亲近欲、胜负欲、认同欲、信仰欲、保护欲、求知欲

## 七、技术栈

- 纯原生 HTML + CSS + JavaScript，无框架，无构建工具
- 数据存储：`localStorage`
- AI：Anthropic API，模型 `claude-sonnet-4-20250514`
- 浏览器直连 API（需要 `anthropic-dangerous-direct-browser-access: true` header）

---

## 八、开发原则（重要）

1. **不要过度工程化**。这是自用/小团队工具，不需要 React、不需要后端、不需要数据库。
2. **瓶颈在 Prompt 质量，不在工程架构**。任何功能改动要先问：这能让文案更好吗？
3. **保持单文件结构**：index.html / style.css / app.js 三个文件，不要拆更多。
4. **localStorage 够用**，不需要云端数据库，除非明确需要多端同步。

## 九、立即要做的事

把当前的单步生成改成**多步引导流程**：

**Step 1 — 选题确认**
AI 根据用户输入的主题，反问几个问题：
- 这个选题对应哪类用户需求？（底层欲望是什么）
- 目标是涨粉/完播/互动？

**Step 2 — 框架选择**
让用户选择内容框架：
- 故事讲述型（情绪回路：温暖→抓强→转折）
- 主张论证型（观点+论据+故事）
- 枚举信息型（好信息罗列）

**Step 3 — 钩子确定**
根据选题和框架，AI 给出3个不同风格的开头钩子供用户选择。

**Step 4 — 生成完整文案**
基于前三步的选择，生成最终文案。比现在的一步到位质量高很多。
