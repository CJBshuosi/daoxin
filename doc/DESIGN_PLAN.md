# 道心文案 · 多步引导流程设计方案

## 一、整体方向

把当前的单步生成改成 **4 步对话式引导流程**，在输出区域用步骤卡片逐步展示。

## 二、四步流程

### Step 1 — 选题确认
- AI 分析用户输入的主题，判断对应的用户需求和底层欲望
- 建议侧重目标（涨粉/完播/互动）
- 用户确认或调整

### Step 2 — 框架选择
- 提供三种内容框架供选择：
  - 故事讲述型（情绪回路：温暖→抓强→转折）
  - 主张论证型（观点+论据+故事）
  - 枚举信息型（好信息罗列）

### Step 3 — 钩子确定
- AI 基于选题+框架，生成 3 个不同风格的开头钩子
- 钩子类型参考：反常识 / 漫热点 / 情感共鸣 / 故事传奇 / 权威数字 / 植入欲望
- 用户选择一个

### Step 4 — 生成完整文案
- 基于前三步的选择，生成最终文案
- 输出：正文文案 + 3条爆款标题 + BGM推荐
- 支持"从 Step 3 重选"回退

## 三、UI 方案

- 步骤卡片渲染在现有输出区域（`#outputArea`）
- 每完成一步才出现下一步（渐进展示）
- 卡片风格与现有 `result-card` 统一
- 支持回退到上一步重新选择
- 不改变整体布局（Header + 侧边栏 + 主区域）

```
输入区（不变）：主题输入 + 生成按钮
    ↓
Step 1 卡片：AI分析 + 确认/调整按钮
    ↓
Step 2 卡片：三选一框架 + 下一步按钮
    ↓
Step 3 卡片：三个钩子选项
    ↓
Step 4 卡片：最终文案（现有结果卡片样式）
```

## 四、记忆系统增强

借鉴 Letta/MemGPT 结构化记忆块思路，将单一 `memory` 字符串拆分为：

```javascript
memory: {
  style: "",      // 风格偏好（AI自动学习）
  audience: "",   // 目标受众特征
  hooks: [],      // 用户钩子类型偏好统计
  history: []     // 最近N次生成摘要
}
```

现有 `banned` 字段保留在 Track 顶层。

## 五、参考来源

- **Mem0**: 分层记忆（用户级/赛道级/会话级）
- **Letta/MemGPT**: 结构化核心记忆块，Agent 自编辑记忆
- **ChatGPT Memory**: 零摩擦自动提取偏好
- **Jasper AI**: Brand Voice + 多步 Campaign 工作流
- **Coze**: 工作流状态管理 + 回退机制

## 六、开发约束

- 保持三文件结构：index.html / style.css / app.js
- 不加框架，纯原生 JS
- 数据继续用 localStorage
- 样式改动尽量少，复用现有 CSS 类
