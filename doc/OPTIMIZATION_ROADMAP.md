# 道心文案优化路线图

> 目标：对标花火Spark，输出包含镜头语言、剪辑节奏、情绪曲线的高质量文案。
> 原则：先软后硬（先方法论后代码），先验证 prompt 效果再改代码。

## 阶段 1：方法论研究（地基）

不写任何代码，纯研究输出文档：

1. 研究短视频爆款方法论 → `doc/knowledge/content-strategy.md`
   - 守正/出奇策略体系（选题角度分类）
   - 每个子方向的选题逻辑和适用场景

2. 研究镜头语言与剪辑节奏 → `doc/knowledge/video-production.md`
   - 口播/图文/混剪的镜头模板
   - 不同时长的节奏模板

3. 研究情绪曲线设计 → `doc/knowledge/emotion-curve.md`
   - 3秒/15秒/60秒的情绪节奏模型

4. 整理垂直赛道知识 → `doc/knowledge/track-国学.md` 等
   - 每个赛道的爆款规律、合规红线

**这个阶段决定了最终输出质量的80%。**

## 阶段 2：重写 Prompt 模板

基于阶段1的知识库：

1. 把知识库浓缩成结构化的 prompt 片段
2. 按新流程重写每个 step 的 system prompt
3. 用同一个主题反复测试对比，调到满意为止
4. 输出 `doc/PROMPT_V2.md`（新版 prompt 设计文档）

可以不改代码，直接在 API playground 里测试 prompt 效果。

## 阶段 3：调整数据结构和流程

prompt 验证通过后，才改代码：

1. **扩展 Track 类型** — 加 targetAudience、product、persona 等画像字段
2. **重设计 Step 流程** —
   ```
   Step 0: 画像收集（首次使用赛道时）
   Step 1: 策略选择（守正/出奇 + 子方向）  ← 替代原来的框架选择
   Step 2: 选题生成（3个选题 + 执行思路）  ← 替代原来的选题分析
   Step 3: 文案大纲（含情绪曲线）          ← 新增
   Step 4: 完整文案 + 拍摄指导             ← 升级
   Step 5: 润色确认                        ← 保留
   ```
3. **更新 API schema** — 新的 Zod schema 适配更丰富的输出格式
4. **更新 store 和组件**

## 阶段 4：UI 适配 + 测试对比

1. 新步骤的 UI 组件
2. 情绪曲线可视化展示
3. 拍摄指导卡片
4. 用花火同样的测试用例对比输出

## 顺序逻辑

```
阶段1（方法论）→ 解决"AI 知道什么"
阶段2（Prompt） → 解决"AI 怎么用知识"
阶段3（代码）   → 解决"产品怎么承载"
阶段4（UI）     → 解决"用户怎么看到"
```
